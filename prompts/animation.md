这个应用是一个记账软件，你可以在app\src\main\java\com\example\jitterpay\MainActivity.kt，当中找到入口。

# 任务

- 请你从 app/src/main/java/com/example/jitterpay/MainActivity.kt 开始，寻找 app/src/main/java/com/example/jitterpay/ui/addtransaction/AddTransactionScreen.kt 页面及其子组件，现在缺少过渡和动画，给这个页面添加过渡和动画，其他的页面暂时不用修改。
- 用户要求动画必须足够丝滑，不能有任何的元素突然出现，如页面的突然全部加载，弹窗的突然出现，突然出现的元素挤开屏幕。这些都会让用户的体验大大下降，这是绝对不允许的。
- 虽然这个任务看起来简单，但是一个优秀的应用都是由各种细节堆砌起来的，这并不简单。

# 要求

- 使用 kotlin compose 推荐的动画、过渡实现，避免使用命令式的动画方式。
- 动画应简洁、大方，不要为了动画而动画，主要是确保视觉上不会有反感，所有的元素都不应该是突然出现的。
- 变量和类命名符合项目规范，不要包含无意义的命名。
- 代码应易读，并在关键位置加上符合最佳实践的注释，避免无意义或浅显的的注释。
- 代码模块化，遵守单一职责原则，避免不相关的代码充斥在同一个文件当中。
- 代码实现遵守最佳实践，你可以运用 context7 以及网络搜索来查找你需要的信息，不要盲目编写，不要猜测。

# 约束

- 不要急于完成，你必须要设置清晰的规划路径，宁可多花一些时间让功能实现更好，也不要给用户一个半成品。
- 规划清晰的执行路径，同时使用你所拥有的工具来提高你的能力范围和开发效率。
- 遵守项目规范和 Kotlin、Android 开发规范，该项目是 Kotlin Compose 项目，遵循其最佳实践。
- 动画代码不建议进行封装，因为代码已经很简洁，不要过度封装，可以直接在 UI 组件当中直接使用。

# 设计思维

- 动画应与不同的 UI 本身相适配，例如竖直排列的设置项，可以在用户向下滚动时，不可见的设置项变为可见时，设置项可以从右侧滑入，并伴随着透明度的变化。不同的 UI 使用不同的动画，不要全部使用渐变，这看起来会十分困惑和单调。
- 动画的核心设计思路就是为了让界面看起来丝滑，不会让用户感觉死板，生硬。

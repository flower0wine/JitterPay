这个应用是一个记账软件，你可以在app\src\main\java\com\example\jitterpay\MainActivity.kt，当中找到入口。

# 任务

- 实现一个用户预算的页面，它支持设置每个月的预算，或者是一个周期内的预算。
- 你需要设计三个页面，第一个用来展示预算，该页面是比较考虑用户视觉的，需要精心打磨。然后就是编辑和添加预算页面，这两个页面基本相似，一些组件可以复用。
- 在 Home 页面的 QuickActions 当中添加该页面的入口。 

# 要求

- 设计要符合APP现有设计，避免主题不一致以及突兀的设计。目前详细的底层功能你不用实现，先设计UI，并且UI组件要模块化，按照现有的页面结构，比如ui/home放页面组件，ui/components/home放置其子UI组件。
- 你需要从用户使用角度来思考页面的设计，思考用户需要什么功能，但是页面又不能太臃肿，需要保持一定的简洁。这是一个非常需要考量的问题。
- 不要使用类似弹窗的形式展示信息，用户很讨厌这样遮挡视野的 UI。

# 约束

- 变量和类命名符合项目规范，不要包含无意义的命名。
- 代码模块化，遵守单一职责原则，避免不相关的代码充斥在同一个文件当中。
- 代码实现遵守最佳实践，你可以运用 context7 以及网络搜索来查找你需要的信息，不要盲目编写，不要猜测。
- 规划清晰的执行路径，同时使用你所拥有的工具来提高你的能力范围和开发效率。
- 不要急于完成，你必须要设置清晰的规划路径，宁可多花一些时间让功能实现更好，也不要给用户一个半成品。

# 用户需求

- 用户有时需要设置一个月的预算，来提醒自己开支不要超过预算，希望有一个比较精美的页面能够帮助自己达成这个目的。
- 用户有时需要反复编辑，添加新的预算，可能是一年的预算或者是一个月，一个星期或一天。
- 用户希望能够在开支达到百分之八十，九十和一百时都发起通知来提醒用户，该提醒默认开启，也可以关闭。
- 用户希望预算能够以进度的方式来展示，如果看起来也很美观，精致的话就更好了。
